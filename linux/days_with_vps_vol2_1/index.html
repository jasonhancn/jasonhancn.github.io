从零开始的VPS生活 Vol2-1：科学上网——轻松搭建Shadowsocks服务端

2017-12-10 Linux运维

shadowsocks大名鼎鼎，是什么、怎么用我就不说了，外事用Google。

一键包很多，但是我个人很喜欢用官方的方法安装，这样更容易更新。

# 切换到repo配置文件夹
cd /etc/yum.repos.d
# 下载ss的copr repo配置
sudo wget -c https://copr.fedorainfracloud.org/coprs/librehat/shadowsocks/repo/epel-7/librehat-shadowsocks-epel-7.repo
# 以下是官网要求安装的包
sudo yum install epel-release -y
sudo yum install gcc gettext autoconf libtool automake make pcre-devel asciidoc xmlto c-ares-devel libev-devel libsodium-devel mbedtls-devel -y
# 相比官网要求，这里还缺一个libsodium.so.13，可能是Vultr的锅，得装
sudo yum install libsodium13 -y
# 更新一下所有包
sudo yum update -y
# 安装ss
sudo yum install shadowsocks-libev -y

修改配置文件/etc/shadowsocks-libev/config.json

{
    "server": ["0.0.0.0","[::]"],
    "server_port":888,
    "password":"password",
    "timeout": 60,
    "method": "chacha20-ietf-poly1305",
    "fast_open": false
}

这个地方增加了一个IPv6访问的功能，理论上也能实现6in4，但是实际操作中浏览器不一定会支持。主要的作用还是可以用IPv6链接服务器。

端口和密码请按照需求修改。随后执行以下命令。

# 开放tcp代理端口
sudo firewall-cmd --permanent --add-port=888/tcp
# 开放udp代理端口
sudo firewall-cmd --permanent --add-port=888/udp
# 防火墙生效
sudo firewall-cmd --reload
# 注意：如果使用1024以下的低端口，需要赋予它绑定低端口的Capacity
sudo setcap cap_net_bind_service=+ep /usr/bin/ss-server
# 启动ss服务
sudo systemctl enable shadowsocks-libev
# 启动ss
sudo systemctl start shadowsocks-libev

在设置了低端口的情况下，因为setcap是对文件生效的，所以如果版本更新了还需要重新设置一次。

因为开启了yum-cron自动更新，所以可能会有第一天好好的第二天就boom了的情况发生，因此可以在/etc/yum/yum-cron.conf的最后加上exclude = shadowsocks*，排除掉自动更新，之后每次手动更新后执行setcap来保证有绑定低端口的权限。

另外一种方法就是用crontab设置一个定时任务，每天凌晨4:30之类的时间自动执行一次setcap，然后重启ss服务 当然，不想这么麻烦直接设置一个1024以上的端口就行了。

Enjoy it~
